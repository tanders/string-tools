string-position   (item string) (string string) &rest key-args &key (start 0) (end (length string)) &allow-other-keys  [Method]

Returns the index of the item string if it occurs completly in string, NIL otherwise. Understands keyargs :start, :end, :from-end :key :test :test-not. :start and :end are related to string.

all-string-positions   (item string) (string string) &rest key-args &key (start 0) accum &allow-other-keys  [Method]

Returns all start positions of item in string if it occurs completly in string, NIL otherwise. Understands keyargs :start, :end, :from-end (?) :key :test :test-not. :start and :end are related to string.

string-find   (item string) (string string) &rest key-args  [Method]

Returns the first found substring in string matching item. Understands keyargs :start, :end, :from-end :key :test :test-not. :start and :end are related to string. Current limitation: the returned substring always has the same length as item, regardless :test, :test-not or :key

string-remove   (item string) (string string) &rest key-args &key (count nil) (end (length string)) &allow-other-keys  [Method]

Returns a string like string but without substrings matching item. If count is given only the first count occurences are removed. Understands keyargs :start, :end, :from-end :key :test :test-not. :start and :end are related to string.

string-count   (item string) (string string) &rest key-args &key (start 0) (end (length string)) &allow-other-keys  [Method]

Returns the number of substrings in string matching item. Understands keyargs :start, :end, :from-end :key :test :test-not. :start and :end are related to string.

find-string-before   (item string) (string string) &rest key-args  [Method]

Returns substring of string before the first substring matching item encounters in string. Understands keyargs :start, :end, :from-end :key :test :test-not. :start and :end are related to string.

read-lisp-file   path  [Function]

Expects a path name to a lisp file and returns the read (but not evaluated) content of the file. For example, if the file contains (1 2 3) then the list (1 2 3) is return (i.e., not a string, but a list, but 1 is not called as a function). Note that only the 1st Lisp form in path is returned.
